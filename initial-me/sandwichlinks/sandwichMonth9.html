Week 3

MockUp For Science Interaction Sites, Sharepoint 2013

Listed below are the key things I was involved with doing this week..

MockUp For Science Interaction Sites
I spent time this week drawing mock-ups for the new set of Science Interaction sites the team will soon be embarking on building. These mock-ups were drawn so that they can be shown to each of the sites (P.S, there are more than 10 sites), and they can then say if they're satisfied with the design or if it needs to be updated with changes here and there.
Sharepoint 2013 On Windows Server 2012 on Virtual Box
This week, I took more time to do more with Sharepoint 2013. I took time to learn about how everything links together in order to be able to develop a Sharepoint solution. So I investigated creating Web Applications, Site Collections, sites, pages, Layouts, content types, columns, etc etc.
Also, due to the nature of how a page can be created by a user in Sharepoint, the typical (shorter route) to create a new page will not ask the user to select a layout (It just uses a default layout) - which isn't what we want in our team, we want the user to be able to select what layout they want...or for the system to at-least be smart enough to guess and use the appropriate layout based on the name that a new page is given. So I took more time to investigate this, firstly looking at how to change a default layout, and then looking at the possibility of using "Sharepoint Content Organiser", and rules and setting up Sharepoint Workflows...in order to be able to dynamically use an appropriate layout based on the naming of a new page.

Week 4

Supporter Of The Week

Supporter Of The Week
It turned out to be a busy of supporting the users of our software. While I was on support, I Worked On An Application That Downloads a BLlob file From Oracle DB, renames it, and sends it to a Sharepoint library, and then update another Sharepoint list that has links that points to the appropriate file in the Sharepoint Library. While doing this, I was able to learn about what DB "Views" are, I also learn about why/when I should or shouldn't consider using them.

I also worked on some legacy Microsoft Access application (with its VB code) that wasn't doing what it was supposed to do only to discover that this was because the Assess application required the user to visit a certain sharepoint site/link that the application was looking to use, and it required the user to first visit the site using Internet Explorer, and provide authentication.

They also worked on debugging and solving some problems that were occurring on some of our live web applications.
