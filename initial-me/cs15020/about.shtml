What I did?

In designing this website, I made sure I followed the requirement specification by making the webpages to be of Doctype, “xHTML 1.0 Strict”. In doing this, I was able to fully design and style the webpages using CSS 3.
On the main page, there is a request for the web user’s name. When the user inputs the make and clicks on submit button, the user would be greeted as appropriate depending on the time of the day. There is also a “declaration of originality” and “Disclaimer” section. I created a page that has a HTML formatted copy of my CV. I used CSS in formatting this page. I also used a menu system in order to navigate around the CV.

How I approached this project?

I started by creating a Cascading Style Sheet that divided the index webpage into sections as appropriate using the DIV tag. The sections included the header, the menu area, the main area and the footer. I designed a banner as an image for the header section; a menu system that linked with my previous work, my blog and my well formatted CV was also created and placed on every page using Server Side Include (SSI). I used JavaScript on the footer sections of the webpages to show when a page was last modified.

Problems you encountered?

I an effort to get the website validated, I struggling in understand and fixing an error that had to do with my webpages being “.shtml” files because I used SSI.

How I solved the problem?

After carefully going through my code, I realised the pages I was including using SSI had tags like HTML and HEAD, after getting rid of this and leaving only the imprtant bit of the code, I revalidated the website and it passed.

Browser versions I used for testing?

I used FireFox, Internet Explorer and Chrome for testing the webpages.
